It is only non-cacheable by default.  Use cache-control to change that.

The Web had a notion of external links, but nobody could agree on the
HTTP syntax so it didn't make the cut.  WebDAV added properties.

Adding a new header field to every GET response would increase latency
for no benefit to the common case and only marginal benefit to the
applications being discussed.  It is therefore more efficient to have
the client request that information only when it is desired.
