> It seems that when you take all the REST constraints as a whole, that
> there is nothing which constrains that all "give me a representation
> of this resource" requests use the same semantic (i.e.  GET in HTTP).
> Tim talks about this[1] as one of his "axioms".

I think it is important to differentiate between things that the
architectural style does not allow (constraints) and things that
it allows but assumes nobody will do.  Tim has a lot of axioms
that are, at best, personal preferences for a rational universe.
Universality is one of them -- the principle that doing it well
enough in one place suggests that nobody else should duplicate
the same functionality, since duplication can be harmful.
